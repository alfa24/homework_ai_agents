### Итерация 6: Транскрибация голосовых сообщений через OpenRouter (whisper-1)

**Подход (KISS):** Голосовое сообщение → скачиваем `.ogg` из Telegram → `client.audio.transcriptions.create(model=whisper-1)` → полученный текст пропускаем через существующий `get_transaction_response_text()`. Отдельный системный промпт не вводим. Работает только через OpenRouter/OpenAI-совместимые провайдеры; Ollama аудио-эндпоинт не поддерживает.

- [x] Обновить `src/handlers.py`: обработчик `F.voice` — скачать ogg, транскрибировать, показать распознанный текст пользователю, далее — существующий текстовый пайплайн (извлечение транзакций + ответ + баланс)
- [x] Логирование: `chat_id` + длительность аудио + распознанный текст + извлечённые транзакции
- [x] Обновить `.env` / `.env.example`: добавить `MODEL_AUDIO=openai/whisper-1`
- [x] Обновить `README.md`: раздел про голосовые сообщения и ограничение по Ollama

**Как протестировать:**

## Sprint 2 — RAG-ассистент на LangChain

**Цель спринта:** добавить параллельную функцию RAG-ассистента по корпусу справочных документов СберБанка (`data/`), не трогая существующий финансовый советник. Команда `/ask <вопрос>` маршрутизирует запрос в RAG; обычный текст, фото и голос по-прежнему идут в финансовый советник. Подробности — `docs/vision.md` §RAG-ассистент (LangChain).
