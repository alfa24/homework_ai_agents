# Workflow (KISS)

Работай только в рамках текущей итерации `docs/tasklist.md`.
Архитектура, стек и качество — по `docs/vision.md` и правилу Conventions.
Не расширяй scope без явного согласия пользователя.

Цикл итерации:
1. **Согласование** — предложи решение и покажи ключевые отрезки кода (или точный список правок по файлам). До явного «ок» не вноси изменения в репозиторий.
2. **Реализация** — после подтверждения внеси изменения в код (и тесты при необходимости).
3. **Пауза** — дождись подтверждения, что результат устраивает.
4. **Тасклист** — обнови `docs/tasklist.md`: чекбоксы текущей итерации; при её завершении — строку в таблице прогресса (статус и короткий комментарий).
5. **Следующая итерация** — переход только после явного согласования.
6. **Коммит** — после завершения цикла и обновления тасклиста сделай коммит с понятным сообщением.

Один цикл за раз. Без лишних файлов и рефакторинга вне согласованного объёма.
