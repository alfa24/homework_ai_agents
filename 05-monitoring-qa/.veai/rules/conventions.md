**Генерация кода**

- Стек и качество: §1 (Python 3.12, aiogram polling, uv, OpenRouter/Ollama через официальный openai client с настройкой провайдера через `base_url`, ruff, pytest). Для RAG-ассистента — LangChain (`langchain`, `langchain-openai`, `langchain-community`, `pypdf`) поверх того же OpenRouter через `base_url`; векторное хранилище — `InMemoryVectorStore`. Мониторинг и evaluation: `langsmith`, `ragas>=0.2.0`, `datasets` (§1, §RAG-ассистент).
- Объём и стиль: §2 — KISS, только нужное для проверки идеи; без преждевременных абстракций и лишних слоёв; сначала рабочий end-to-end, затем точечные улучшения.
- Размещение и ООП: §3 — `src/`, `tests/`, `docs/`; **один класс = один файл**; простой OOP.
- Слои и поток: §4 — handler → сервис → LLM-клиент; конфигурация и логирование — отдельные инфраструктурные модули.
- Данные: §5 — минимальная модель MVP, без БД; контекст диалога только как ограниченный in-memory буфер при необходимости.
- LLM: §6 — один настраиваемый клиент/модель из конфига; при сбоях — безопасный ответ пользователю, детали только в логах.
- Конфиг: §8 — единый `Settings` из окружения; обязательные параметры валидировать при старте.
- Логи: §9 — стандартный `logging`, stdout, без секретов и чувствительных значений.
- Сборка и запуск: §10 — не усложняй контракты `make` / uv / Docker без необходимости.
