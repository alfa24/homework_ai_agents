# Идея: Персональный финансовый советник

## Описание проекта

Персональный финансовый советник реализован в виде Telegram-бота на базе LLM-ассистента.

## Основная задача

Бот ведет учет доходов и расходов пользователя на основе сообщений в чате, автоматически извлекая из них структурированные данные о финансовых транзакциях. Предоставляет отчеты о балансе по запросу.

## Ключевые особенности

- **Платформа**: Telegram
- **Функционал**: 
  - Ведение таблицы учета доходов и расходов
  - Автоматическое извлечение данных о транзакциях из текстовых сообщений
  - Обработка изображений (чеки, скриншоты выписок банков); для отладки vision — синтетические скриншоты в `debug_bank_statements/` (корень `live/`, RU/EN, макеты: приложение, пуш-карточки, «PDF»-выписка)
  - Обработка голосовых сообщений: транскрибация речи в текст и последующее извлечение транзакций по тому же пайплайну, что и для текстовых сообщений
  - Формирование отчетов о балансе
- **Технологии**: 
  - LLM со structured output для обработки текстовых сообщений
  - VLM для обработки изображений
  - Speech-to-Text (whisper-1 через OpenRouter) для транскрибации голосовых сообщений
  - Поддержка переключения между внешними и локальными моделями

## Расширение: RAG-ассистент

В рамках Sprint 2 бот получает дополнительную функцию — RAG-ассистента на базе LangChain, который отвечает на вопросы по корпусу справочных документов СберБанка. Функция работает параллельно с финансовым советником, не заменяя его.

**Корпус документов (`data/`):**
- `ouk_potrebitelskiy_kredit_lph.pdf` — общие условия потребительского кредитования (PDF)
- `usl_r_vkladov.pdf` — условия по вкладам (PDF)
- `sberbank_help_documents.json` — 212 вопросов-ответов справки по дебетовым и кредитным картам

**Ключевые особенности RAG-ассистента:**
- Взаимодействие через команду `/ask <вопрос>` (команда `/ask_reset` сбрасывает историю RAG-диалога). Обычные сообщения, фото и голосовые по-прежнему идут в финансовый советник.
- Индексация: `/index` пересобирает векторный индекс, `/index_status` показывает текущее состояние. Индекс строится один раз при старте.
- Векторное хранилище: `InMemoryVectorStore` (LangChain) — без внешней БД, всё в памяти процесса.
- Пайплайн ответа: query transformation с учётом истории диалога → retriever top-K → ответ LLM по извлечённому контексту (`rag_query_transform_chain`, см. `docs/references/naive-rag.ipynb`).
- Модели и параметры (`MODEL_CHAT_RAG`, `MODEL_EMBEDDINGS`, `RETRIEVER_K`, `CHUNK_SIZE`, `CHUNK_OVERLAP`, `DATA_DIR`) задаются через `.env`; провайдер — OpenRouter через `base_url` и официальный openai-клиент внутри LangChain.

## Расширение: мониторинг и оценка качества

В рамках Sprint 3 бот получает инструменты для контроля качества RAG-пайплайна: отображение источников в ответах, автоматический трейсинг запросов через LangSmith, синтез QA-датасетов из корпуса документов и оценка качества ответов через RAGAS-метрики (`faithfulness`, `answer_relevancy`, `context_precision` и др.).
